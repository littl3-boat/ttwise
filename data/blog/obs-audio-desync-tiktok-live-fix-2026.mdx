## 为什么标准的“同步”逻辑在 TikTok 上会失效

当你向 Twitch 直播时，OBS 会将视频和音频打包成一个流。但在 TikTok 上，许多创作者使用 OBS 虚拟摄像头将视频发送到 TikTok Live Studio，同时通过单独的虚拟音频线（Virtual Cable）发送音频。这创建了两条独立的路径，具有不同的延迟。

## 同步检查清单

如果你正在经历这种偏移，请按顺序执行以下步骤。

### 第一步：匹配你的采样率
导致“漂移”式不同步（随时间推移而恶化）的最常见原因是采样率不匹配。
- [ ] 打开 Windows 声音设置（控制面板）。
- [ ] 确保你的麦克风、扬声器和虚拟音频线都设置为 **48000Hz**。
- [ ] 打开 OBS 设置 → 音频，确保它也设置为 **48kHz**。

### 第二步：计算延迟量
不要靠猜。使用“拍手测试”。
1. 开启你的 TikTok 直播（或使用 TikTok Live Studio 中的本地录制功能）。
2. 对着镜头拍一下手。
3. 观看回放。如果声音在你的手接触*之前*发生，说明音频超前了。
4. 估算差距（通常在 200ms 到 400ms 之间）。

### 第三步：应用 OBS 偏移量
由于视频较慢，我们必须放慢音频以匹配视频。
1. 在 OBS 中，查看 **混音器**。
2. 点击三个点（齿轮图标），选择 **高级音频属性**。
3. 找到 “同步偏移 (ms)” 列。
4. 为你的麦克风和桌面音频输入一个正值（例如 `250`）。
   *   *注意：从 200ms 开始，并以 50ms 为增量进行微调。*

### 第四步：创建“TikTok 专用”配置
由于这个偏移会破坏你在 Twitch 或 YouTube 的同步，你应该在 OBS 中创建一个单独的 **配置（Profile）**。
1. 前往顶部菜单：**配置 → 新建**。
2. 命名为 “TikTok Live”。
3. 在这里应用你的同步偏移。当你不播 TikTok 时，切换回你的 “Twitch” 配置。

## 决策指南：虚拟摄像头 vs. 推流码

| 设置方法 | 不同步风险 | 同步复杂度 |
| :--- | :--- | :--- |
| **虚拟摄像头 + 虚拟音频线** | **高** | 需要手动设置偏移（200ms-500ms）。 |
| **RTMP 推流码（直连）** | **低** | 通常会自动保持同步。 |
| **TikTok Live Studio（原生）** | **无** | 无需 OBS，但功能有限。 |

## FAQ：解决奇特的极端情况

**“我的音频刚开始是同步的，但一小时后就开始漂移。为什么？”**
这几乎总是 CPU 散热限频问题或采样率不匹配（见第一步）。如果你的 CPU 达到 100%，它会优先处理视频帧而“丢弃”音频定时数据。

**“我需要虚拟音频线（VB-CABLE）吗？”**
是的。如果你使用 OBS 虚拟摄像头，你*必须*使用虚拟音频线将 OBS 的音频导入 TikTok Live Studio。否则，你可能只是在 TikTok 中选择了原始麦克风，这会绕过你所有的 OBS 滤镜和同步设置。

**“我能在 TikTok Live Studio 而不是 OBS 中修复这个问题吗？”**
TikTok Live Studio 有音频输入的“延迟”设置，但不如 OBS 精确。最好在源头（OBS 高级音频属性）进行修复。

## 总结

“TikTok 不同步”并不是 OBS 的 Bug，它是使用两个不同的软件层来处理一个流的副产品。通过承认虚拟摄像头的处理延迟并应用 **200ms–300ms 的正向偏移**，你可以获得那种专业的、音画合一的观感，避免观众流失。
