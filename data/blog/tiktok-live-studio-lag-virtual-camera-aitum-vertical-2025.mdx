## 为什么虚拟摄像头在 TikTok Live Studio 中会卡顿？

当你使用 **OBS → 虚拟摄像头 → TikTok Live Studio** 这一链路时，实际上创建了一个多阶段的处理流水线：

1. **OBS** 采集并合成场景。
2. **Aitum Vertical** 创建竖屏画布。
3. **OBS 虚拟摄像头** 将画布输出为摄像头信号。
4. **TikTok Live Studio** 接收信号并进行最终推流编码。

掉帧通常发生在第 3 或第 4 步：

- **虚拟摄像头编码负载**：OBS 输出虚拟摄像头本身需要消耗资源。
- **Live Studio 处理延迟**：TikTok 软件接收虚拟摄像头信号时的重采样开销。

## 关键修复方案

### 1. 强制匹配分辨率与帧率
如果你的 OBS 画布是 1080x1920 60fps，但 TikTok 设置为 30fps，系统会进行实时的帧率转换，这在 Mac 或低配 PC 上会导致显著卡顿。
- **确保三者统一**：OBS 画布、虚拟摄像头输出、TikTok 推流设置，必须完全一致。

### 2. 降低虚拟摄像头分辨率
不要在虚拟摄像头上追求极致清晰度。
- 尝试将 OBS 虚拟摄像头输出设置为 **720x1280**。
- TikTok Live Studio 可以轻松将 720p 信号上采样，而这能显著降低编码压力。

### 3. 避免编码器竞争
- **OBS 设置**：如果 TikTok Studio 正在使用 NVENC（硬件编码），请尝试将 OBS 的虚拟摄像头输出设置为 **x264（CPU）**，避免两个程序抢占同一个显卡编码芯片。

### 4. Mac 用户特别注意
Mac 版 TikTok Live Studio 仍处于优化阶段。
- 确保已授予 OBS 和 Live Studio 的 **“摄像头权限”**。
- 尝试关闭 OBS 的预览窗口（Hide Preview），以释放显存资源。

---

## 最终检查清单

- [ ] **OBS 预览流畅**（确认源头没问题）。
- [ ] **虚拟摄像头在其他应用（如 Zoom）中流畅**（排除驱动问题）。
- [ ] **分辨率完全匹配**（1080p vs 1080p）。
- [ ] **帧率锁定一致**（统一 60fps 或 30fps）。
- [ ] **关闭多余后台程序**（特别是 Chrome 等占用 GPU 的应用）。

如果以上操作后依然卡顿，建议申请 **TikTok 推流码（Stream Key）**。通过 RTMP 直接从 OBS 推流可以绕过整个虚拟摄像头环节，这是解决卡顿的最彻底方案。
