## TL;DR
- **核心痛点**：OBS 预览画面丝滑流畅，但 TikTok 直播端（通常通过虚拟摄像头或 Live Studio）却像幻灯片一样卡顿。
- **搜索意图**：解决 OBS 预览不卡但 TikTok 直播卡顿、帧率低的问题。
- **关键结论**：预览流畅是因为它仅代表本地渲染。当 GPU 负载接近 100% 时，系统会牺牲后台推流软件的性能。必须“以管理员身份运行 OBS”并限制游戏 FPS。

## 幻灯片式的直播体验？

这是直播中最让人沮丧的时刻：你看着副屏，OBS 显示 60 帧，画面完美；你玩着游戏，体感也极其流畅。

但当你打开手机看自己的直播间，或者看到弹幕里刷：
*“主播在放幻灯片吗？”*
*“这画面卡得像在烤面包机上玩。”*

你感到困惑：*“为什么我的 OBS 显示 0 丢帧，直播间却卡成狗？”*

这种情况通常不是网络问题，而是你电脑内部的一场资源“无声战争”。在 2025 年，如果你在推 TikTok Live（尤其是用 **TikTok Live Studio** 配合 **OBS 虚拟摄像头**），你实际上是在同时运行两个繁重的广播软件。

大多数创作者认为卡顿 = 网络差。但在这种特定情况下，通常是 **渲染延迟** 或 **编码过载**。

当你的 GPU 占用率达到 99% 时，Windows 会自动降低后台应用的优先级。如果 OBS 只能抢到一点点 GPU 残余性能，它处理预览（Preview）没问题，但却无法稳定地将每一帧图像传输给虚拟摄像头或推流后端。

### 1. 以管理员身份运行 OBS（必做！）
这是解决 90% OBS 卡顿问题的终极方案。
*   **原理**：Windows 有个“游戏模式”，默认把所有 GPU 性能给当前游戏。
*   **做法**：关闭 OBS，右键图标选择“以管理员身份运行”。
*   **效果**：Windows 会将 OBS 视为高优先级进程，即便游戏想吃满 100% 性能，系统也会强行给 OBS 留出一部分 GPU 资源。

### 2. 限制游戏内的帧率（FPS）
如果你不设防，游戏会尝试跑到 200+ FPS，吃光所有 GPU 资源，让 OBS 连“合成画面”的性能都没有。
*   **做法**：在游戏设置里，将帧率锁定在直播帧率的倍数（如 60, 120 或 144）。
*   **目标**：保持 GPU 占用率在 90% 以下。

### 3. 匹配帧率设置
一个常见的错误是 OBS 设置了 60 帧，但 TikTok Live Studio 设置了 30 帧。这种帧率不匹配会导致视觉上的“卡顿感”，即便技术上没有丢帧。
*   **检查**：确保 OBS 和 TikTok Live Studio 的输出 FPS 设置完全一致。

### 4. 关闭“双重预览”
如果你通过虚拟摄像头把 OBS 画面传给 TikTok Live Studio：
1.  在 TikTok Live Studio 里添加源。
2.  **隐藏或最小化** TikTok Live Studio 的预览窗口。
因为同时渲染两套预览窗口（OBS 一套，TTLS 一套）是非常浪费资源的。

## 总结

你不需要昂贵的双机推流方案也能流畅推 TikTok。你只需要尊重 GPU 的极限。通过管理员权限运行和锁定游戏帧率，你能确保 OBS 有足够的“呼吸空间”来处理那些丝滑的画面。
