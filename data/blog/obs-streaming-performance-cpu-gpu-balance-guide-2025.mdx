## TL;DR
- **核心痛点**：游戏画面流畅但直播间卡顿（或反之），无法平衡 CPU 和 GPU 负载导致性能浪费或推流崩溃。
- **搜索意图**：优化 OBS 直播性能，解决 CPU/GPU 负载不均导致的掉帧问题。
- **关键结论**：直播性能的核心是“预留余量”。通过强制限制游戏帧率（FPS Cap）来避免 GPU 满载，优先选择硬件编码器（如 RTX 显卡的 NVENC）。

## 直播性能的“木桶效应”

在单机直播中，你的电脑就像一个同时处理多项任务的工厂。如果游戏占用了 99% 的资源，那么负责推流的“车间”（OBS）就会因为缺乏原材料（系统资源）而停工，导致直播间出现掉帧。

想要获得完美的直播效果，关键不在于你的配置有多高，而在于你如何**平衡** CPU 和 GPU 的负载。

1.  **锁定游戏帧率（必做）**：不要让游戏跑无限制的帧率。如果你的显示器是 144Hz，建议锁定在 120 或 144；如果是 60Hz，锁定在 60。这能为编码器留出宝贵的 GPU 时间。
2.  **优先使用硬件编码器**：对于 NVIDIA 用户，首选 `NVENC`。它有专门的硬件单元处理编码，几乎不占用游戏性能。
3.  **关闭不必要的后台程序**：尤其是开启了“硬件加速”的浏览器或 Discord，它们会偷偷吃掉 GPU 性能。
4.  **以管理员身份运行 OBS**：这会让 Windows 优先处理 OBS 的渲染请求。

### 场景 A：游戏流畅，直播卡顿
这通常意味着 **GPU 满载** 或 **编码器压力过大**。
*   **对策**：降低游戏画质（尤其是光追、抗锯齿），或者在 OBS 中降低输出分辨率（如从 1080p 降至 720p）。

### 场景 B：直播流畅，游戏卡顿
这通常意味着 **CPU 占用过高**。
*   **对策**：如果你在使用 x264（软编），请尝试切换到硬件编码。同时关闭不必要的 CPU 密集型插件（如复杂的实时美颜或 AI 降噪）。

## 监控核心指标

在 OBS 中打开 `视图` -> `统计`，重点关注以下两个数值：
*   **平均渲染时间**：理想情况下应小于 **3ms**。如果超过 5ms，说明 GPU 压力太大。
*   **由于编码延迟跳过的帧**：如果这个数字在跳动，说明你的编码器（NVENC 或 x264）跟不上。

## 总结

直播不是一场关于最高 FPS 的竞赛，而是一场关于**稳定性**的马拉松。通过合理设置帧率上限和选择正确的编码器，你可以在不增加硬件支出的情况下，让直播间观众获得丝滑的观看体验。
